## Context Block for Discovery & Research Agents

**Audience & Scope:** This newsletter targets a **broader, non-specialist audience** interested in AI agents and coding tools—not just developers. Prioritize accessibility and real-world applicability over technical depth.

**Primary Focus:** Emphasize **agent harness patterns and coding architectures** (tabs 11–18) as the core narrative. The Qwen/Ollama/Unsloth setup content (tabs 1–10) should serve as *supporting context* for how local LLMs integrate into agent systems, not the main story.

**Key Problem:** Center the newsletter around **building reliable, long-running agents**—reliability, robustness, and production patterns are the throughline. Use content on effective harnesses, agent sandboxes, and engineering best practices to frame this challenge.

**Grouping Strategy:** Organize by **pattern/concept first, then by tool/platform.** Group Anthropic, OpenAI, and open-source approaches (OpenHarness, OpenCode) side-by-side to show how different teams solve the same reliability problem. Use the local LLM content to illustrate cost/control trade-offs in agent deployment.

**Tone:** Focus on lessons learned and practical patterns, not promotional comparisons. Flag any concrete architectural decisions or trade-offs (e.g., sandbox vs. local inference).
